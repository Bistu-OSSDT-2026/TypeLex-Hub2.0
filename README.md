# TypeLex-Hub2.0
## 小组：Team-22 | 开源软件开发技术课程实训项目

## 一、项目概述
TypeLex-Hub2.0 是一款面向在校学生、英语入门学习者的轻量化单词打字练习工具，整合**网页交互、单词记忆存储、词法拼写校验**三大核心模块。
项目采用纯前端网页为主、配套词法解析脚本为辅的架构，无需安装客户端、无需部署后端服务，仅通过浏览器即可完整运行，适配Windows系统，满足日常背单词、打字测速、拼写纠错三类学习需求。

### 开发背景
在英语学习过程中，单纯背诵单词缺少打字实操训练，市面上工具大多功能繁杂、需要登录注册。本项目轻量化设计，本地离线可用，自带学习向页面背景，降低新手使用门槛，同时作为课程开源实训项目，完整遵循Git团队协作、开源贡献流程。

## 四、项目完整目录结构

项目主要由 `homepage`、`memory`、`spell` 和 `README.md` 四部分组成。

1. **homepage 文件夹**

`homepage` 是项目的前端交互主页模块，主要用于展示单词打字练习的入口页面。

其中，`index.html` 是单词打字练习主页，`汪汪主页.png` 是网页背景图片资源。

2. **memory 文件夹**

`memory` 是单词记忆与本地存储模块，主要用于保存用户练习记录和单词学习数据。

其中，`words` 文件夹用于存放内置单词词库文件，`index.html` 是单词记忆页面，`script.js` 负责本地存储逻辑，`style.css` 负责页面样式。

3. **spell 文件夹**

`spell` 是词法解析和拼写校验核心模块，主要用于识别单词拼写错误并提供纠错提示。

其中，`public/assets` 和 `src` 用于存放页面资源和核心源码，`index.html` 和 `standalone.html` 用于页面运行，`package.json` 和各类编译配置文件用于模块配置。

4. **README.md 文件**

`README.md` 是项目说明文档，主要用于介绍项目功能、运行方式、目录结构、协作规范和开源声明。

## 五、核心功能介绍

1. **单词打字测速练习**

随机加载内置英语单词，用户输入完成后实时统计打字速度、正确率，直观展示练习效果。

2. **本地记忆存储**

利用浏览器本地存储持久保存练习记录，下次打开页面时可以查看历史练习数据，实现记忆复习闭环。

3. **智能词法拼写校验**

调用 `spell` 模块的词法解析能力，自动识别单词拼写错误，给出纠错提示，帮助用户巩固单词正确写法。

4. **学习向自定义页面背景**

页面内置手绘学习主题背景，整体风格较柔和，适合长时间学习，也支持后续替换为自定义背景图片。

5. **离线全功能支持**

项目核心逻辑运行在本地浏览器中，无需联网、无需登录、无广告，能够保护用户学习隐私。

## 六、开源团队协作与贡献规范
### 1. 分支开发规范
- `main` 分支：仅存放稳定、可正常运行的正式代码，禁止直接修改提交；
- 新增功能、bug修复必须新建独立功能分支，开发完成后提交Pull Request合并。

### 2. Git 提交注释规范
统一格式：`类型: 修改内容简述`
- `feat:` 新增功能、页面、资源
- `fix:` 修复页面报错、逻辑bug
- `docs:` 修改README、补充文档说明
- `style:` 调整页面样式、更换背景图

### 3. PR合并审核规则
1. 提交PR时需标注关联修改内容；
2. 至少一名小组组员完成代码审阅；
3. 页面、功能改动需本地验证可正常运行，方可合并至main分支；
4. 新增功能必须同步更新README文档。

### 4. 新手友好贡献入口（无代码门槛）
1. 完善README中英文部署、操作教程；
2. 优化网页背景样式，增加文字半透明遮罩提升可读性；
3. 扩充内置单词词库，新增四六级、中考词汇；
4. 补充常见报错FAQ、完善使用说明；
5. 复现页面bug并提交标准化Issue反馈。
